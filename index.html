🐧 Windows에서 리눅스 실행 방식 비교

🖥️ Hyper-V 방식
하드웨어 (CPU, 메모리, 디스크)
↑
Hyper-V 하이퍼바이저
↑
Windows OS
↑
Linux VM (완전한 독립 OS)
작동 원리: 하드웨어 가상화 기술을 사용하여 완전히 독립적인 가상 머신을 생성합니다. 각 VM은 자체 커널과 전체 OS를 실행합니다.

장점

  • 완전한 격리
  • 다양한 OS 지원
  • 보안성 우수

단점

  • 높은 리소스 사용
  • 부팅 시간 길음
  • 성능 오버헤드
📦 Windows 컨테이너 방식
하드웨어
↑
Windows OS 커널
↑
컨테이너 런타임 (Docker)
↑
Linux 컨테이너들
(커널 공유, 프로세스 격리)
작동 원리: OS 수준 가상화로 호스트 커널을 공유하면서 프로세스와 파일시스템을 격리합니다. 가벼운 가상화 방식입니다.

장점

  • 빠른 시작 속도
  • 낮은 리소스 사용
  • 쉬운 배포

단점

  • 커널 공유 제약
  • 격리 수준 낮음
  • 호환성 제한
⚡ WSL (Windows Subsystem for Linux)
하드웨어
↑
Windows NT 커널
↑
WSL Translation Layer
↑
Linux 환경 (네이티브 바이너리)
(시스템 콜 변환, 경량 VM)
작동 원리: WSL2는 경량화된 VM에서 실제 Linux 커널을 실행하며, Windows와 긴밀하게 통합됩니다. 시스템 콜을 빠르게 변환합니다.

장점

  • Windows와 통합
  • 빠른 성능
  • 낮은 오버헤드

단점

  • GUI 제한적
  • 일부 기능 미지원
  • Windows 전용

특성 Hyper-V Windows 컨테이너 WSL

시작 시간 느림 (분) 빠름 (초) 매우 빠름 (즉시)
